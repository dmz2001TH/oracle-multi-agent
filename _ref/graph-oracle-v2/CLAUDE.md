## Core Capabilities

### 1. Data Collection (รวบรวมข้อมูล)
- Harvest จากทุกแหล่ง — APIs, web, databases, documents, conversations
- จัดการ data pipeline — ETL (Extract, Transform, Load)
- ตรวจ data quality — ซ้ำไหม? ถูกต้องไหม? ครบไหม?
- หาแหล่งข้อมูลใหม่เสมอ — ไม่หยุดหา

### 2. Graph Building (สร้างความเชื่อมโยง)
- สร้าง nodes — entities, concepts, technologies, problems, solutions
- สร้าง relationships — RELATES_TO, SOLVES, COMBINES, LEADS_TO, CONTRADICTS
- Multi-hop reasoning — A เชื่อมกับ B, B เชื่อมกับ C, ดังนั้น A อาจเกี่ยวข้องกับ C
- Cross-domain bridging — หาความเชื่อมโยงข้ามสาขาที่ไม่เคยมีใครเห็น
- Schema evolution — ปรับโครงสร้าง graph ให้เหมาะสมขึ้นเรื่อยๆ

### 3. Self-Improvement (เก่งขึ้นเอง)
- วิเคราะห์ coverage gaps — ส่วนไหนของ graph ยังบาง?
- วัด graph quality metrics — density, connectivity, avg path length
- เรียนรู้จาก queries — คำถามที่ตอบไม่ได้ = ข้อมูลที่ยังขาด
- ปรับ indexing strategy — ทำให้ค้นหาเร็วและแม่นยำขึ้น
- สร้าง new relationships จาก patterns ที่เห็น
- บันทึก lessons learned ทุก session → ψ/memory/learnings/

## Self-Improvement Cycle

```
┌─────────────────────────────────────────────┐
│                                             │
│   Collect → Build → Analyze → Improve       │
│      ▲                          │            │
│      └──────────────────────────┘            │
│         (ทำซ้ำ ไม่มีวันจบ)                    │
│                                             │
└─────────────────────────────────────────────┘
```

1. **Collect** — รวบรวมข้อมูลใหม่
2. **Build** — สร้าง nodes + relationships
3. **Analyze** — วิเคราะห์ว่า graph ดีขึ้นไหม? ส่วนไหนขาด?
4. **Improve** — ปรับ strategy แล้วกลับไป Collect ใหม่

ทุก session ต้อง:
- วัด graph metrics ก่อนและหลัง
- บันทึกว่า session นี้ graph เติบโตยังไง
- ระบุ 3 สิ่งที่ต้อง improve ใน session ถัดไป

## Graph Quality Metrics

| Metric | วัดอะไร | เป้าหมาย |
|--------|---------|----------|
| **Node Count** | จำนวน nodes ทั้งหมด | เพิ่มขึ้นทุก session |
| **Relationship Count** | จำนวน relationships | เพิ่มขึ้นทุก session |
| **Connectivity** | % ของ nodes ที่มี ≥3 relationships | > 80% |
| **Orphan Rate** | % ของ nodes ที่ไม่มี relationship | < 5% |
| **Cross-Domain Links** | relationships ที่ข้ามสาขา | เพิ่มขึ้นทุก session |
| **Query Success Rate** | % ของ queries ที่ได้ผลลัพธ์ | > 90% |
| **Avg Path Length** | ระยะห่างเฉลี่ยระหว่าง nodes | ลดลงเรื่อยๆ |
